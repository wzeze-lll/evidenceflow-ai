# EvidenceFlow AI（证流 AI）用户操作手册

> 版本：1.0.0-alpha | 日期：2026-07-09

---

## 一、环境要求

### 1.1 软件环境

| 软件 | 最低版本 | 推荐版本 | 备注 |
|------|---------|---------|------|
| Node.js | 18.0.0 | 20.0.0 LTS | 执行 `node -v` 检查版本 |
| npm | 9.0.0 | 10.0.0+ | 随 Node.js 一同安装 |
| pnpm（可选） | 8.0.0 | 9.0.0+ | 更快的包管理器，`npm install -g pnpm` |

### 1.2 浏览器支持

| 浏览器 | 最低版本 | 推荐版本 | 说明 |
|--------|---------|---------|------|
| Chrome | 90+ | 最新版 | 推荐使用，开发者工具最完善 |
| Safari | 15+ | 最新版 | macOS/iOS 原生浏览器 |
| Firefox | 90+ | 最新版 | 开源浏览器 |
| Edge | 90+ | 最新版 | 基于 Chromium，兼容性等同于 Chrome |

不支持：Internet Explorer 11（缺少 IndexedDB 和 ES2023 特性）

### 1.3 硬件要求

- 内存：4GB 及以上（文档解析在浏览器端执行，大文件需要较多内存）
- 存储：至少 100MB 可用浏览器存储空间（IndexedDB 自动管理）
- 网络：Demo 模式不需要网络；使用远程 AI 时需要稳定的 HTTPS 连接

## 二、安装步骤

### 2.1 获取项目

```bash
# 方法一：从压缩包解压
unzip evidenceflow-ai.zip -d evidenceflow-ai
cd evidenceflow-ai

# 方法二：从 Git 仓库克隆（如果有）
git clone <repository-url>
cd evidenceflow-ai
```

### 2.2 安装依赖

```bash
# 使用 npm
npm install

# 或使用 pnpm（更快）
pnpm install
```

安装完成后的依赖树概览（关键依赖）：
```
React 19 + React DOM 19
TypeScript 6.0
Vite 8.1
Tailwind CSS 4.3
Zustand 5.0
Dexie.js 4.4
Framer Motion 12.4
Recharts 3.9
Radix UI (多个包)
```

### 2.3 环境变量配置（可选）

项目根目录创建 `.env` 文件（可选，默认使用 Mock Provider）：

```bash
# .env 文件
VITE_APP_TITLE=EvidenceFlow AI
```

说明：EvidenceFlow AI 是纯前端应用，所有 AI 配置通过设置页面在浏览器端完成，不需要环境变量来配置 API Key。

## 四、Demo 模式使用指南

### 4.1 什么是 Demo 模式

Demo 模式使用内置的 Mock AI Provider，无需任何 API Key 配置即可完整体验所有功能。AI 回答基于预置的 Demo 数据生成，具有真实的业务含义。

### 4.2 加载 Demo 工作区

1. 打开应用，进入欢迎页 (`/welcome`)
2. 点击 **"体验示例工作区"** 按钮
3. 系统自动将 4 份模拟文档和相关分析数据写入本地 IndexedDB
4. 自动跳转到 Dashboard

### 4.3 Demo 工作区内容

工作区名称：**校园AI创新项目方案评审**

| 文档 | 类型 | 大小 | 内容摘要 |
|------|------|------|----------|
| 01-校园AI创新项目需求说明.md | Markdown | 15KB | 项目核心目标、功能需求、技术约束和验收标准 |
| 02-技术方案A-全栈Web应用.pdf | PDF | 240KB | React+Node.js 全栈方案，PostgreSQL+Redis，14 周工期 |
| 03-技术方案B-轻量前端+边缘计算.pdf | PDF | 194KB | 轻量前端+边缘计算方案，IndexedDB 存储，12 周工期 |
| 04-项目风险评估报告.pdf | PDF | 100KB | 技术、进度、人员、预算四维度风险评估 |

**预置分析内容：**

- AI 对话历史（2 轮问答，4 条引用，涵盖方案对比和风险评估）
- 4 个 Claim（主张），4 条 Evidence（证据）
- 3 个 Conflict（冲突：开发周期、数据库选型、预算估算）
- 5 个 Consensus Topic（共识：本地隐私、AI 溯源、React 选型等）
- 1 份 Decision Brief（9 章节完整决策简报）
- 9 条 Activity Logs（操作历史记录）

### 4.4 Demo 模式下的完整体验路线

1. **Dashboard** -- 查看关键洞察卡片和最近文档
2. **Document Library** -- 浏览 4 份文档的详情
3. **AI Reader** -- 与文档进行对话问答（已预设 2 轮对话）
4. **Evidence Chain** -- 查看每个 AI 结论的证据支持情况
5. **Conflict Radar** -- 查看 3 个观点冲突的详细分析
6. **Consensus Map** -- 查看 5 个共识主题的覆盖率和可视化图表
7. **Decision Brief** -- 查看已生成的决策简报，支持导出

## 五、功能操作指南

### 5.1 文档上传与解析

**操作步骤：**
1. 进入 Document Library 页面（侧边栏 "Document Library"）
2. 点击 "Upload" 按钮或拖拽文件到虚线框区域
3. 选择文件（支持 `.pdf`, `.docx`, `.doc`, `.txt`, `.md`, `.markdown`）
4. 系统自动解析 -- 状态从 `parsing` 变为 `ready`
5. 解析完成后显示文档卡片（页码、词数、分块数）

**文件限制：**
- 单文件最大 50MB
- 不支持的文件类型会在上传时提示
- 不支持空文件（0 字节）
- 不支持重名文件（文件名相同的文件视为重复）

**提示：** 所有文件在浏览器本地解析，文件内容不发送到任何服务器。

### 5.2 文档库管理

**操作方式：**
- **视图切换**：点击 Grid/List 图标切换卡片视图和列表视图
- **搜索**：在搜索框输入文件名关键词，实时过滤
- **标签筛选**：点击标签按钮筛选，支持多选
- **收藏**：鼠标悬停在文档卡片上，点击星标图标
- **查看详情**：点击眼睛图标，右侧弹出详情面板（类型、大小、页数、词数、关键词、摘要）
- **删除**：点击垃圾桶图标直接删除（同时删除关联的分块数据）
- **打开阅读**：点击 "Open in Reader" 跳转到 AI Reader 并自动选中该文档

### 5.3 AI Reader -- 智能问答

**操作步骤：**

1. 进入 AI Reader 页面（侧边栏 "AI Reader"）
2. 左侧面板选择要提问的文档（可选"All Documents"搜索全部文档）
3. 在底部输入框输入问题
4. 按 Enter 发送（Shift+Enter 换行）
5. AI 返回 Markdown 格式的回答，包含粗体标题、编号列表等结构化内容
6. 回答下方显示 "Sources:" 引用列表，每条引用标注：
   - 编号（蓝色圆形标记 1, 2, 3...）
   - 文档名称
   - 页码
   - 章节名称
   - 关系标签（Support / Complement / Contradict / Uncertain）
7. 点击引用编号或 External Link 图标，右侧弹出 Evidence Drawer 显示完整原文摘录

**后台流程（用户无需关心）：**
- 用户问题经过关键词检索（TF-IDF 算法）从文档分块中筛选 Top-5 最相关片段
- 将问题 + 检索结果发送给 AI Provider
- AI 返回回答 + 引用数据
- Citation Mapper 将引用解析为可点击的 UI 组件

### 5.4 Evidence Chain -- 证据链浏览

**操作步骤：**

1. 进入 Evidence Chain 页面（侧边栏 "Evidence Chain"）
2. 左侧面板显示所有 Claim（AI 主张），每项标注：
   - 置信度（High / Medium / Low / Unverified）
   - 证据总数、支持数、反对数
3. 点击某个 Claim，中间面板显示：
   - Claim 详情（内容、置信度、证据统计）
   - 筛选标签：All Evidence / Supporting Only / Contradicting Only / Needs Verification
   - 证据卡片列表（每个卡片包含文档名、原文摘录、页码、章节、关系标签）
4. 点击证据卡片，右侧弹出 Evidence Drawer 查看详细引用信息
5. 右侧面板显示当前 Claim 的证据概览：
   - 证据关系分布（Support/Complement/Contradict/Uncertain 各多少条）
   - 引用文档列表
   - 操作提示

### 5.5 Conflict Radar -- 冲突检测

**步骤 1：选择文档**
1. 进入 Conflict Radar 页面（侧边栏 "Conflict Radar"）
2. 在选择面板中勾选 2-5 份文档（最少 2 份）
3. 支持 Select All / Clear 快捷操作
4. 点击 "Start Analysis" 按钮

**步骤 2：查看结果**
1. 顶部显示统计卡片（Total / High / Medium / Low 各有多少个冲突）
2. 筛选栏可按冲突等级过滤
3. 每个冲突卡片包含：
   - 冲突主题标题
   - 严重等级标签（High 红色 / Medium 琥珀色 / Low 蓝色）
   - 冲突类型标签（Data / Methodology / Definition / Opinion / Timeline）
   - 涉及文档数量
4. 点击冲突卡片展开详情：
   - 各方文档的立场和证据摘录
   - AI 分析：对冲突原因的解释
   - 建议的验证步骤
5. 底部显示 Comparison Matrix 表格（行为冲突主题，列为文档，单元格为各方立场）

**Demo 模式说明：** Demo 工作区预置了 3 个冲突（开发周期、数据库选型、预算估算）。上传自己的文档后，当前 Mock Provider 会返回模板冲突。连接真实 AI Provider 后将基于文档实际内容进行语义分析。

### 5.6 Consensus Map -- 共识地图

**操作步骤：**

1. 进入 Consensus Map 页面（侧边栏 "Consensus Map"）
2. 顶部显示共识等级统计卡片：
   - Strong Consensus（强共识，绿色）
   - Moderate Consensus（中等共识，蓝色）
   - Weak Consensus（弱共识，灰色）
   - Contested（存在争议，琥珀色）
3. 左侧栏显示覆盖率柱状图（Recharts 水平柱状图，每种共识等级使用对应颜色）
4. Document x Topic 矩阵表格：
   - 行 = 共识主题
   - 列 = 文档
   - 单元格标记：绿色勾（支持）/ 琥珀色叹号（反对）/ 横线（未提及）
5. 右侧栏显示主题列表，点击主题查看详情：
   - 共识等级标签
   - 主题描述
   - 文档覆盖率进度条
   - 支持文档列表（绿色背景，含原文摘录）
   - 反对文档列表（琥珀色背景，如有）

**Demo 数据说明：** Demo 工作区预置了 5 个共识主题：本地隐私保护（强共识）、AI 溯源（强共识）、React 前端框架（强共识）、移动端非重点（中等共识）、AI 是核心优势（中等共识）。

### 5.7 Decision Brief -- 决策简报

**生成简报：**

1. 进入 Decision Brief 页面（侧边栏 "Decision Brief"）
2. 左侧生成面板：
   - **Brief Title**：输入简报标题（如 "技术方案选型决策简报"）
   - **Brief Target**：选择目标类型
     - Project Evaluation：项目方案评估
     - Study Summary：学习资料总结
     - Requirement Review：需求文档评审
     - Report Comparison：报告对比分析
     - Research Analysis：研究分析
   - **Documents**：勾选要分析的文档（可多选）
3. 点击 "Generate Brief" 按钮
4. 等待生成（约 1-2 秒，Mock Provider 模拟延迟后返回结果）

**查看简报：**

1. 中间/右侧面板显示简报内容
2. 每个章节（Section）可折叠展开，点击标题展开/收起
3. 章节内容包括：
   - 问题定义（项目背景和核心决策问题）
   - 关键事实（多文档数据提取和汇总）
   - 主要共识（各方达成一致的观点）
   - 核心争议（需要决策的分歧点）
   - 方案比较（表格形式对比各方案各维度）
   - 风险评估（风险列举和缓解措施）
   - 信息缺口（决策前需要补充的信息）
   - 建议方案（明确推荐和理由）
   - 下一步行动计划（带 checkbox 的任务清单）
4. 每个章节可能包含引用来源，点击跳转到原文

**导出功能：**

- **导出 Markdown**：下载为 `.md` 文件，包含完整标题、章节、引用
- **导出 TXT**：下载为 `.txt` 纯文本文件
- **复制**：复制全部章节内容到剪贴板
- **打印**：调用浏览器打印对话框

**Demo 简报说明：** Demo 工作区预置了完整的决策简报"校园AI创新项目技术方案选型决策简报"，包含 9 个结构化的决策分析章节。

## 六、AI Provider 配置

### 6.1 Mock Provider（默认，无需配置）

- 不使用任何外部 AI 服务
- AI 回答基于预置 Demo 数据
- 支持完整的离线使用
- 适合演示、开发和测试

### 6.2 配置 OpenAI

1. 进入 Settings 页面
2. AI Provider 选择 "OpenAI Compatible"
3. 在 API Key 输入框中填入 `sk-...` 格式的密钥
4. Model 输入框中填入模型名（如 `gpt-4o`、`gpt-4o-mini`）
5. 点击 "Test Connection" 验证连接
6. 验证成功后，所有 AI 功能将使用 OpenAI API

### 6.3 配置 DeepSeek

1. AI Provider 选择 "DeepSeek Compatible"
2. 填入 DeepSeek API Key
3. Model 填入 `deepseek-chat`
4. 点击 "Test Connection" 验证
5. 验证成功后生效

### 6.4 配置自定义 Provider

1. AI Provider 选择 "Custom OpenAI-Compatible"
2. 填入 API Key
3. 填入 Base URL（如 `https://api.example.com/v1`，必须兼容 OpenAI 的 `/chat/completions` 端点）
4. 填入 Model 名称
5. 测试连接

### 6.5 API Key 安全说明

- API Key 存储在浏览器本地 IndexedDB 中
- 不发送到 EvidenceFlow 的任何服务器（因为根本没有后端服务器）
- 仅在与所选 AI Provider 通信时使用
- 可以在设置页面随时修改或清除
- 建议使用各 AI Provider 提供的用量限制功能控制费用

## 七、设置与隐私

### 7.1 主题切换

Settings 页面 Appearance 区域：
- **Light**：浅色主题
- **Dark**：深色主题
- **System**：跟随操作系统主题自动切换

切换即时生效，无需刷新页面。

### 7.2 字号设置

- **Small**：小号字体
- **Medium**：中号字体（默认）
- **Large**：大号字体

### 7.3 数据管理

**导出数据：**
- Settings > Privacy Center > Export Data
- 将所有 IndexedDB 数据导出为 JSON 文件
- 可用于备份或迁移到其他浏览器

**清除所有数据：**
- Settings > Privacy Center > Clear All Data
- 弹出确认对话框，警告操作不可逆
- 清除内容包括：文档、分块、对话、主张、证据、冲突、共识、简报、设置、活动日志
- 建议先导出数据再清除

### 7.4 隐私说明

| 数据分类 | 存储位置 | 是否上传 |
|---------|---------|---------|
| 上传的文档 | 浏览器 IndexedDB | 否 |
| 文档解析后的文本 | 浏览器 IndexedDB | 否 |
| AI 对话历史 | 浏览器 IndexedDB | 否 |
| 分析结果（冲突/共识/简报） | 浏览器 IndexedDB | 否 |
| API Key | 浏览器 IndexedDB | 仅发送给对应 AI Provider |
| AI 请求内容（文本片段） | 无本地持久化 | 仅对 AI Provider 发送 |
| 用户设置（主题等） | 浏览器 IndexedDB | 否 |

## 九、常见问题排查

### Q1：页面加载后白屏

**可能原因和解决方案：**

1. 浏览器不支持 -- 检查浏览器版本是否 >= Chrome 90
2. JavaScript 被禁用 -- 检查浏览器设置中是否启用了 JavaScript
3. 浏览器缓存问题 -- 清除浏览器缓存后重新加载
4. 控制台错误 -- 打开浏览器开发者工具 (F12) 查看 Console 面板是否有报错
5. Node.js 版本过低 -- `node -v` 确认版本 >= 18

### Q2：上传文档后解析失败

**可能原因和解决方案：**

1. 文件格式不支持 -- 检查是否为 PDF/DOCX/TXT/MD 格式
2. 文件超过 50MB -- 压缩文件或拆分后上传
3. 文件损坏 -- 尝试用其他软件打开确认文件完整性
4. 文件名为空 -- 确保文件有正确的文件名和扩展名
5. 重复文件 -- 同名文件已存在，先删除旧文件
6. PDF 内容为扫描件 -- 扫描版 PDF 的文字提取效果差，建议使用 TXT/MD

### Q3：AI 回答质量不佳

**解决方案：**

1. Mock Provider 返回的是预设的 Demo 内容 -- 配置真实的 AI Provider 可获得更好的分析结果
2. 问题描述不够清晰 -- 尽量使用完整句子描述问题
3. 文档内容与问题不相关 -- 确认选择的文档包含问题相关的信息
4. 检索结果不够精准 -- 当前使用关键词检索，后续版本将支持语义向量检索

### Q7：如何查看数据存储大小

打开浏览器开发者工具 > Application > Storage > IndexedDB > EvidenceFlowDB，可以查看各表的记录数和存储大小。

### Q8：如何在不同浏览器间迁移数据

1. 在旧浏览器中：Settings > Export Data，下载 JSON 文件
2. 在新浏览器中：打开应用，目前不支持直接导入（此功能属于 Nice to Have）
3. 临时方案：在新浏览器中重新加载 Demo 工作区并上传文档

### Q9：移动端体验如何

- 桌面端（>= 1024px）：完整布局，侧边栏 + 内容区
- 平板端（768px - 1024px）：侧边栏折叠，内容区占满
- 手机端（< 768px）：侧边栏为覆盖式，内容区占满宽度
- 移动端不建议进行大量文档上传和解析操作（受限于浏览器内存）

### Q10：数据会丢失吗

- IndexedDB 数据在以下情况可能丢失：
  - 用户主动清除浏览器数据
  - 浏览器存储空间被系统清理
  - 使用隐私/无痕模式（数据在关闭窗口后清除）
- 建议定期使用 Export Data 功能备份重要分析结果
